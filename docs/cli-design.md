# CLI Design

Agents Team is intended to be used as a CLI.

## Command Shape

```bash
agents-team <command> [arguments] [options]
```

The initial command surface should be small and stable:

```bash
agents-team list
agents-team validate
agents-team doctor [tool]
agents-team render <tool> [agent]
agents-team install <tool>
agents-team update <tool>
```

Where `<tool>` is one of:

```text
codex
claude
opencode
all
```

## `list`

Lists canonical agents discovered under `agents/`.

Expected output should include:

- Agent id.
- Description.
- Enabled targets.

## `validate`

Validates canonical agent files and adapter compatibility.

It should check:

- Markdown frontmatter syntax.
- Required fields.
- Permission schema.
- Target names.
- Adapter support.
- Generated filename collisions.

Validation should distinguish errors from warnings.

Errors block installation. Warnings should be visible but should not block unless
the user passes a future strict flag.

## `doctor`

Runs a read-only diagnostic of the repository, target tools, and installation
state.

Examples:

```bash
agents-team doctor
agents-team doctor all --project .
agents-team doctor codex
```

Expected checks:

- Python version.
- `agents/` directory exists.
- Canonical agents validate.
- Target outputs render.
- Global or project target directories resolve.
- Target CLI commands are available on `PATH`, when applicable.
- Installation dry-run detects files that would be created, updated, or skipped.

`doctor` should exit with a non-zero code only for errors. Missing external CLI
commands or existing manual files are warnings, because the generator may still
be usable.

## `render`

Renders generated output without installing it.

Examples:

```bash
agents-team render codex
agents-team render codex --root-agent orchestrator
agents-team render claude --root-agent orchestrator
agents-team render claude reviewer
agents-team render opencode builder --out generated/
```

Expected behavior:

- If no agent is provided, render all enabled agents for the selected tool.
- If an agent is provided, render only that agent.
- If `--out` is provided, write generated files there.
- If `--out` is omitted and one agent is selected, print the generated file.
- If `--out` is omitted and multiple agents are selected, print filenames and
  contents in a readable format or require `--out`.
- If `--root-agent <id>` is provided for Codex, render `AGENTS.md` root
  instructions from that agent instead of a subagent TOML file.
- If `--root-agent <id>` is provided for Claude Code, render `CLAUDE.md`
  main-session instructions from that agent instead of a subagent file.

## `install`

Installs generated files into the target tool directory.

Examples:

```bash
agents-team install codex
agents-team install codex --root-agent orchestrator
agents-team install claude
agents-team install claude --root-agent orchestrator
agents-team install opencode
agents-team install all
```

Global installation is the default.

Project installation is explicit:

```bash
agents-team install all --project .
agents-team install codex --project C:\path\to\project
```

## `update`

Updates installed files from the current repository state.

Examples:

```bash
agents-team update all
agents-team update codex
agents-team update codex --root-agent orchestrator
agents-team update claude --root-agent orchestrator --project .
```

For MVP, `update` may share implementation with `install`.

Once manifests exist, `update` should use them to report changed, unchanged,
created, and removed files.

## Installation Locations

### Global

```text
Codex:       ~/.codex/agents/
Codex root:  ~/.codex/AGENTS.md
Claude Code: ~/.claude/agents/
Claude Code root: ~/.claude/CLAUDE.md
OpenCode:    ~/.config/opencode/agents/
```

### Project

```text
Codex:       <project>/.codex/agents/
Codex root:  <project>/AGENTS.md
Claude Code: <project>/.claude/agents/
Claude Code root: <project>/CLAUDE.md
OpenCode:    <project>/.opencode/agents/
```

## Flags

### `--project`

Installs into a project-local agent directory instead of the global directory.

```bash
agents-team install all --project .
```

### `--dry-run`

Shows planned file writes without writing files.

```bash
agents-team install all --dry-run
```

### `--backup`

Creates backups before overwriting existing files.

```bash
agents-team install all --backup
```

Backups are not created by default.

### `--force`

Allows overwriting files even if they do not look generated by Agents Team.

```bash
agents-team install codex --force
```

This should be used carefully and should not be required for normal updates.

## Exit Codes

Suggested exit codes:

- `0`: success.
- `1`: validation or installation error.
- `2`: invalid CLI usage.
