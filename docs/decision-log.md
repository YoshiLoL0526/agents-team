# Decision Log

This file records decisions that define the project direction.

## 2026-04-20: Canonical Format

Decision: Agents Team will use Markdown with YAML frontmatter as its canonical
agent format.

Rationale:

- Prompts are naturally Markdown.
- Structured metadata can be validated.
- Claude Code and OpenCode already use Markdown frontmatter.
- Codex can be generated from this source into TOML.

## 2026-04-20: Native Target Formats

Decision: Generated files will use each tool's native format.

Mappings:

- Codex: TOML.
- Claude Code: Markdown with YAML frontmatter.
- OpenCode: Markdown with YAML frontmatter.

Rationale:

- The installed output should be idiomatic for each tool.
- The repository should not force a target tool to consume non-native files.

## 2026-04-20: Global Installation Default

Decision: Installation is global by default.

Rationale:

- The main workflow is to keep a personal agent team available everywhere.
- Project-local installation is still needed and will be explicit through
  `--project`.

## 2026-04-20: Project Installation Flag

Decision: Project-local installation will use a `--project` flag.

Example:

```bash
agents-team install all --project .
```

Rationale:

- Avoids accidental writes into a project when the user expects global setup.
- Keeps command shape simple.

## 2026-04-20: Detailed Permissions From The Start

Decision: The canonical schema will include detailed permissions immediately.

Rationale:

- Agent tools differ most sharply around permissions and tool access.
- Delaying this would make early agents harder to migrate safely.
- A conservative permission layer gives adapters a clear translation target.

## 2026-04-20: Shared Prompt With Optional Overrides

Decision: Each agent should have one shared prompt body, with optional
tool-specific overrides.

Rationale:

- The team should behave as consistently as possible across Codex, Claude Code,
  and OpenCode.
- Tool-specific differences should be exceptional and explicit.

## 2026-04-20: Initial Agent Team

Decision: The first canonical team includes `orchestrator`, `explorer`,
`builder`, `reviewer`, and `researcher`.

Rationale:

- These roles cover coordination, codebase inspection, implementation, review,
  and internet research.
- The team is small enough to validate the source format and adapters without
  over-specializing too early.

## 2026-04-20: Safe Overwrite Default

Decision: Install and update overwrite only files that contain the Agents Team
generated-file marker. Files without the marker are skipped unless `--force` is
passed.

Rationale:

- This protects manually authored tool agents.
- It keeps normal updates simple for files generated by this repository.
- `--backup` is available when the user wants a copy before replacement.
